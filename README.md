# Lib.MathCore

## Опис

Бібліотека математичних операцій для Mini-ChatGPT.
Використовується для обробки логітів, обчислення функції втрат та вибору токенів.

## Класи

IMathOps інтерфейс для використання бібліотеки  
SoftmaxCalculator обчислює ймовірності з логітів  
LossCalculator обчислює функцію втрат  
ProbabilitySampler виконує вибір індексу  
MathOpsImpl реалізує інтерфейс IMathOps  

## Методи

Softmax(ReadOnlySpan<float> logits) перетворює логіти у ймовірності  
CrossEntropyLoss(ReadOnlySpan<float> logits, int target) обчислює помилку  
ArgMax(ReadOnlySpan<float> scores) повертає індекс максимуму  
SampleFromProbs(ReadOnlySpan<float> probs, Random rng) повертає випадковий індекс  

## Очікування інтеграції

Бібліотека використовується через інтерфейс IMathOps  
На вхід подаються логіти або ймовірності  
На вихід повертаються ймовірності або індекси токенів  
# TinyNN

## 1. Головна ідея

Модель приймає масив ID токенів - context. Кожен ID - це номер токена у словнику
Самі ID не мають змісту, це просто індекси. За ними модель знаходить відповідні вектори
Цей масив і є поточний контекст, з якого модель буде робити прогноз

## 2. Embeddings і hidden

За кожним ID береться його embedding-вектор. Embeddings - це таблиця, де кожному токену відповідає свій набір чисел
Ці вектори складаються разом і діляться на кількість токенів. У результаті отримується hidden-вектор
Hidden - це наближене представлення сенсу поточного контексту

## 3. OutputWeights і OutputBias

Hidden передається далі в лінійний шар. OutputWeights - це матриця параметрів для всіх токенів
Кожен стовпець цієї матриці відповідає одному токену і визначає, наскільки він підходить до hidden
OutputBias - це додатковий зсув для кожного токена. Він додається до результату незалежно від hidden

## 4. Logits

Hidden перемножується з OutputWeights. Для кожного токена обчислюється своє значення
Потім до кожного значення додається OutputBias. У результаті отримується масив logits
Logits - це оцінки того, наскільки кожен токен підходить до поточного контексту

## 5. Softmax і probs

Logits передаються в softmax. Softmax перетворює їх у probs. Probs - це ймовірності вибору кожного токена
Кожне значення знаходиться між 0 і 1. Сума всіх значень дорівнює 1

## 6. Target і loss

Target - це правильний наступний токен. Це просто ID токена, який має йти після контексту
З probs береться значення за індексом target. Чим воно більше, тим краще модель вгадала
Loss - це число, яке показує помилку моделі. Чим він менший, тим краще працює модель

## 7. Градієнт і оновлення

З probs обчислюється градієнт. Він показує, як змінити параметри, щоб зменшити loss
Для target значення буде від’ємне, тому його оцінка буде збільшуватись
Для інших токенів значення додатні, тому їх оцінки будуть зменшуватись
Далі оновлюються Embeddings, OutputWeights і OutputBias. Це і є процес навчання


## TinyTransformer

### 1. Головна ідея
Модель приймає масив ID токенів (context). На відміну від базової TinyNN, TinyTransformer використовує механізм самовзаємодії (self-attention). Це дозволяє моделі краще "розуміти" ширший контекст та складні зв'язки між словами на різних позиціях, а не просто усереднювати їх.

### 2. Embeddings та позиція
Кожен ID токена перетворюється на вектор ознак (`wte` - word token embeddings). Оскільки трансформери обробляють усі токени одночасно (а не по черзі), до цих векторів обов'язково додається інформація про їхню позицію в тексті (`wpe` - word position embeddings), щоб модель знала порядок слів.

### 3. Механізм Attention
Вектори проходять через механізм уваги за допомогою матриць Query, Key та Value (`wq`, `wk`, `wv`). Модель обчислює "вагу" (важливість) кожного токена відносно всіх інших у поточному контексті. Далі результат нормалізується та проходить через шар проекції (`wo`).

### 4. Лінійні шари (Feed Forward)
Після механізму уваги дані нормалізуються (`ln1g`) та проходять через повнозв'язну нейронну мережу (матриці `w1`, `w2` та зсуви `b1`, `b2`) з функцією активації. Це дозволяє моделі витягувати глибші закономірності з тексту.

### 5. Logits та обрізання контексту
Останній шар генерує `logits` — масив оцінок для кожного слова зі словника (`vocabSize`). Важлива особливість: модель має суворе обмеження на розмір вхідного тексту (`ContextSize`). Якщо переданий контекст довший за це значення, модель автоматично відкидає найстаріші токени (бере лише кінець масиву), щоб уникнути помилок переповнення.

### 6. Генерація (Без TrainStep)
Отримані `logits` сумісні з інтерфейсом `IMathOps` (зокрема `SoftmaxCalculator`), де вони перетворюються на ймовірності (`probs`) для вибору наступного токена. На поточному етапі розробки TinyTransformer реалізує лише прямий прохід (forward pass) для генерації тексту і, на відміну від TinyNN, **не має** методу `TrainStep` для навчання.

## Очікування інтеграції (Як використовувати моделі B5)

**1. Створення та конфігурація**
Для ініціалізації моделей B5 необхідний `vocabSize` (розмір словника, отриманий від підсистеми даних B1). 
Для ініціалізації TinyTransformer використовується клас `TinyTransformerConfig`, де додатково вказуються:
* `EmbeddingSize` — розмірність векторів (наприклад, 16).
* `ContextSize` — максимальна кількість токенів, які модель може обробити за раз.
* `HeadCount` — кількість голів механізму уваги.

**2. Робота з Checkpoints**
Обидві моделі реалізують спільний контракт `ILanguageModel` та підтримують збереження/завантаження стану. 
Для серіалізації використовуються фабрики `TinyTransformerModelFactory` та `TinyNNModelFactory`. Вони дозволяють конвертувати модель у `Payload` (зберігаючи конфігурацію та всі матриці ваг) і відновлювати її без втрати точності логітів.
