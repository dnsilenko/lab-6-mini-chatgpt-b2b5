# Lib.MathCore

## Опис

Бібліотека математичних операцій для Mini-ChatGPT.
Використовується для обробки логітів, обчислення функції втрат та вибору токенів.

## Класи

IMathOps інтерфейс для використання бібліотеки  
SoftmaxCalculator обчислює ймовірності з логітів  
LossCalculator обчислює функцію втрат  
ProbabilitySampler виконує вибір індексу  
MathOpsImpl реалізує інтерфейс IMathOps  

## Методи

Softmax(ReadOnlySpan<float> logits) перетворює логіти у ймовірності  
CrossEntropyLoss(ReadOnlySpan<float> logits, int target) обчислює помилку  
ArgMax(ReadOnlySpan<float> scores) повертає індекс максимуму  
SampleFromProbs(ReadOnlySpan<float> probs, Random rng) повертає випадковий індекс  

## Очікування інтеграції

Бібліотека використовується через інтерфейс IMathOps  
На вхід подаються логіти або ймовірності  
На вихід повертаються ймовірності або індекси токенів  
# TinyNN

## 1. Головна ідея

Модель приймає масив ID токенів - context. Кожен ID - це номер токена у словнику
Самі ID не мають змісту, це просто індекси. За ними модель знаходить відповідні вектори
Цей масив і є поточний контекст, з якого модель буде робити прогноз

## 2. Embeddings і hidden

За кожним ID береться його embedding-вектор. Embeddings - це таблиця, де кожному токену відповідає свій набір чисел
Ці вектори складаються разом і діляться на кількість токенів. У результаті отримується hidden-вектор
Hidden - це наближене представлення сенсу поточного контексту

## 3. OutputWeights і OutputBias

Hidden передається далі в лінійний шар. OutputWeights - це матриця параметрів для всіх токенів
Кожен стовпець цієї матриці відповідає одному токену і визначає, наскільки він підходить до hidden
OutputBias - це додатковий зсув для кожного токена. Він додається до результату незалежно від hidden

## 4. Logits

Hidden перемножується з OutputWeights. Для кожного токена обчислюється своє значення
Потім до кожного значення додається OutputBias. У результаті отримується масив logits
Logits - це оцінки того, наскільки кожен токен підходить до поточного контексту

## 5. Softmax і probs

Logits передаються в softmax. Softmax перетворює їх у probs. Probs - це ймовірності вибору кожного токена
Кожне значення знаходиться між 0 і 1. Сума всіх значень дорівнює 1

## 6. Target і loss

Target - це правильний наступний токен. Це просто ID токена, який має йти після контексту
З probs береться значення за індексом target. Чим воно більше, тим краще модель вгадала
Loss - це число, яке показує помилку моделі. Чим він менший, тим краще працює модель

## 7. Градієнт і оновлення

З probs обчислюється градієнт. Він показує, як змінити параметри, щоб зменшити loss
Для target значення буде від’ємне, тому його оцінка буде збільшуватись
Для інших токенів значення додатні, тому їх оцінки будуть зменшуватись
Далі оновлюються Embeddings, OutputWeights і OutputBias. Це і є процес навчання
